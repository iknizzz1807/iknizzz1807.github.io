AIC Multimodal Search: A Hybrid Video, Audio & Image Engine

When text-based search engines fail: Architecting a sub-second retrieval pipeline blending vector embeddings (Milvus) and lexical search (Elasticsearch) for massive unstructured media archives.

Tech Stack Python, FastAPI, Milvus, Elasticsearch, OpenCV, FFmpeg
Role Backend Systems Architect

The Chaos of Archiving Media Content

A significant problem facing media agencies, production houses, and large content publishers is the inability to locate specific moments within petabytes of video footage. Tagging video and audio files manually is labor-intensive and prone to human error. Standard relational databases (SQL) or text-based search engines (Solr) are fundamentally incapable of searching for a "red sports car driving in the rain" unless a human has explicitly typed those words into the video's metadata.

The client needed a system capable of accepting natural language queries and returning the exact timestamp of a video where that action occurred—instantaneously.

Architecting a Hybrid Retrieval System

To solve this, I designed the AIC Multimodal Search Engine. A purely vector-based semantic search struggles with exact keyword matching (e.g., specific names, serial numbers), while a purely lexical search fails at semantic understanding. The solution required a "hybrid" retrieval pipeline:

  • Python (FastAPI) as the API Gateway: A high-throughput REST API server to receive raw media files, segment them, and manage the indexing queues asynchronously.
  • Milvus (Vector Database): The core engine for semantic understanding. Milvus stored high-dimensional embeddings generated from video frames and audio transcripts, allowing the system to mathematically compare the visual and auditory content of a video clip to the user's text query.
  • Elasticsearch (Lexical Search): Maintained an index of any extracted metadata, OCR text, and exact keywords to ensure that a search for a specific entity name didn't get lost in the vector space math.

The Ingestion Pipeline Implementation

The ingestion process was the heaviest computational burden. I developed asynchronous background workers in Python. When a video file was uploaded, the pipeline employed FFmpeg to extract the audio track and segment the video into manageable chunks (e.g., 5-second intervals). Concurrently, OpenCV sampled keyframes from the video stream.

These frames were pushed through a Vision-Language Model (VLM, like CLIP or an equivalent multimodal model) to generate semantic vector embeddings. Simultaneously, the audio was passed through an Automatic Speech Recognition (ASR) model (Whisper) to generate text transcripts. The resulting vectors were pushed to Milvus, and the text was indexed in Elasticsearch. The system unified the scoring of both databases (Reciprocal Rank Fusion) before returning the top results.

Business Impact

By automating the indexing of unstructured media through computer vision and NLP models, the system completely eliminated the need for human tagging. The hybrid architecture (Milvus + Elasticsearch) achieved sub-second query response times and unparalleled accuracy. Users could now instantly locate specific 5-second clips buried deep within thousands of hours of historical footage.
